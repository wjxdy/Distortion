# 《失真 Distortion》项目约定

## 背景
本项目用 **Godot 4**（客户端，`Game/client`）+ **Node.js**（LLM 代理后端，`Game/server`）。
项目负责人是 **Vibe Coding**：不直接看/写 Godot 代码，主要靠**在编辑器里拖拽、可视化调整**来做界面。所以下面的规则是硬性的。

---

## ⭐ 核心规则：能用编辑器节点，就绝不用脚本动态生成

写任何**和场景/界面相关**的 Godot 内容时：

- **静态结构必须写进 `.tscn`，作为真实节点**——背景、立绘、按钮、标签、面板、输入框、气泡、特效图、Timer、HTTPRequest 等，凡是"摆在那儿"的东西，都要是编辑器场景树里看得见、**能拖位置、能拉大小、能改属性**的节点。
- **禁止用脚本 `XxxNode.new()` + `add_child()` 去创建本应摆在界面上的控件**（按钮/标签/面板等）。这样的节点编辑器里看不见、拖不动，违背本项目工作方式。
- **脚本只负责逻辑和动态部分**：信号连接、状态机、数据请求、打字机/动画 tween、按运行时数据切换的内容（如对话气泡填字、情绪立绘切帧、临时横幅）。
- 脚本通过 `@onready var x: Type = $节点路径` 引用 `.tscn` 里的节点，而不是自己造。

一句话：**摆着的 → 进 `.tscn`（可拖）；动起来的逻辑 → 进脚本。**

---

## 改动 `.tscn` 时的注意事项

- 改 `.tscn` 或 `project.godot` 前，若 Godot 编辑器正开着该场景，改完要提醒用户 **Reload Saved Scene（场景→重新加载已保存的场景）**，否则编辑器会用旧版覆盖改动。
- 不要让用户在编辑器里手动 Ctrl+S 覆盖刚改过的 `.tscn`。
- 编辑器有时会自动给节点加 `uid`/`unique_id`、或删 `project.godot` 里的某些默认值，属正常，不必反复纠正。

## 美术与占位

- 美术风格：**8 比特复古像素**；字体：Zpix（`fonts/zpix.ttf`，全局默认）。
- 尚无美术的场景/物件，先用**纯色 `ColorRect` 方块 + 文字标签**占位（同样作为 `.tscn` 节点），后期用户再替换为真图。

## 实现与验证（用 godot-mcp）

- **实现/调试 Godot 功能时，优先用 `godot-mcp`**（已配为用户级 MCP）：运行项目、捕获运行时报错/调试输出、读取场景树与项目结构、必要时创建场景/加节点。
- **改完要用 godot-mcp 自验**改动能否跑通、有没有报错，而不是只把验证甩给用户 F5。
- **兜底**：若本会话 godot-mcp 未加载或未连上（MCP 工具仅在会话启动时加载；无头/后台运行可能没有），退回命令行无头校验，并明确告知用户用的是兜底方式：
  - 加载场景无报错：`Godot --headless --path . res://scenes/<场景>.tscn --quit-after 6`
  - 确定性逻辑测试：`Godot --headless --path . -s res://tests/run_tests.gd`
- 用 godot-mcp 建/改 `.tscn` 后，提醒用户在编辑器 **Reload Saved Scene** 才能看到。
- 涉及真实 UI 交互/手感的改动，最终仍由用户在编辑器 F5 实机确认。
