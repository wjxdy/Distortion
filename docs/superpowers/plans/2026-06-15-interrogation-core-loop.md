# 《失真 Distortion》MVP 游戏流程设计（Godot 自研用）

> 这是一份**流程/内容设计**，不是代码计划。描述 MVP 每个阶段在干什么、要做成什么、做到什么算完成。引擎：Godot 4，赛博朋克像素风（参照《2064: Read Only Memories》）。

**整体结果（一句话）：** 一个 5–8 分钟、能完整玩一遍的赛博像素 demo——开场世界观 → 真·AI 审讯撞墙 → 探索取证 → 撬出第一块真相 → 结尾钩子。投资人/评委玩一遍就懂：这游戏是什么、独特在哪、为什么想看完整版。

**贯穿全程只有一个真正的"系统"（对话+判定），其余每段都是线性演出。**

---

## 贯穿系统 · 对话 + 判定（唯一的硬骨头，优先做）

- **老人 = 大模型**：接腾讯云大模型。prompt 写死他的假记忆人格（坚信孤身一人、家人被 AI 害死、AI 有罪）+ "会不经意漏出温暖细节但绝不主动说破"的钩子 + 记住对话历史。
- **钥匙 / 钩子 / 真相 状态**：游戏记录"已持有哪些线索、已揭开哪些真相"。玩家发言**命中关键词 + 已持有对应钥匙**时，才撬开一条真相。
- **判定刻意做成确定性**（不靠 AI 现场裁判）：保证演示稳定、可复现；老人的"撒谎/临场应答"交给 AI，既惊艳又不失控。
- **完成标准**：能真实自由对话；没钥匙问不出真相、有钥匙问对必出真相。

---

## 阶段 0 · 开场序章（约 1 分钟）

- **目的**：立世界观 + 交代老人是谁 + 抛钩子。
- **玩家体验**：黑屏冷开——苍老声音"我老伴怎么还没回来？"，平静的 AI 声音圆谎"她出去了，很快回来。"→ 像素图一张张闪过：赛博街景、人人低头问 AI、广告牌「有问题？问就对了」、**一则泛黄旧新闻「周明远 创造首个 AI 大模型」+ 他年轻意气风发的照片** → 侦探独白"这世上，只剩我还会怀疑它们。"
- **要做成**：一串像素图片（素材包 / AI 生成）+ 文字 + 淡入淡出 / 打字机效果 + 背景音。
- **完成标准**：从启动顺畅播到推门进审讯室，氛围到位。
- **反差即钩子**：刚见过万人敬仰的"AI 之父"，下一秒他就是审讯桌对面坚信"AI 害了我全家"的糊涂老人。

## 阶段 1 · 审讯室 · 初次对话（撞墙）

- **目的**：亮出核心玩法，让玩家产生"他记忆不对劲"的疑问。
- **玩家体验**：像素审讯室，侦探与老人对坐，底部输入框。玩家自由打字，老人由大模型实时回答——平静否认有家人，偶尔漏出"她做的红烧肉"又愣住带过。
- **要做成**：审讯室像素背景 + 老人立绘 + 对话 UI（输入框 + 文本框）+ 接腾讯云大模型。
- **完成标准**：能真实来回聊，且没拿到钥匙时他死活不吐真相。

## 阶段 2 · 探索取证（拿钥匙）

- **目的**：B 循环的另一半——离开对话桌去找证据。
- **玩家体验**：触发"查阅周明远自制 AI 的记忆档案残片" → 看到残片《2019.5.20：妻子 林秀兰 · 蓝裙子》→ 获得线索（钥匙）。
- **要做成**：一个极简"档案/证物"面板（点开显示残片文字 + 一张像素图）+ "已获得线索"提示。
- **完成标准**：玩家拿到钥匙，游戏记住"已持有"。

## 阶段 3 · 审讯室 · 破真相（勾中钩子）

- **目的**：闭环高潮——带钥匙回来，问对地方，撬开真相。
- **玩家体验**：玩家用线索提问（"蓝裙子那个人，她是谁？"）→ 老人记忆裂开 → 真相碎片浮现「林秀兰，他妻子，2021 病逝——不是 AI」→ 老人动摇。
- **要做成**：真相碎片演出（裂痕 / 卡片弹出 + 音效，可简单）+ 触发判定（持钥匙 + 命中关键词）。
- **完成标准**：没钥匙问不出、有钥匙问对必出——可控、可复现。

## 阶段 4 · 结尾钩子（约 30 秒）

- **目的**：收束 demo + 勾完整版。
- **玩家体验**：真相揭开后，侦探独白/字幕"这……已经是这个月第三个了。（待续）"。
- **要做成**：一段结尾文字演出 + 黑屏 / 标题。
- **完成标准**：玩家看完有"想知道后面"的冲动。

---

## 建议制作顺序（5 天，先难后易、先核心后皮）

1. **D1–D2 · 对话+判定系统（核心，先做掉风险）**：接通腾讯云大模型 → 老人能自由对话且撒谎 → 加钥匙/钩子/真相状态 → 跑通"撞墙→（给钥匙）→问对→出真相"。**先用最朴素的 UI 验证逻辑，能玩通即可。**
2. **D3 · 阶段 2 探索面板**：把"探索拿钥匙"接进闭环，B 循环完整成立。
3. **D3–D4 · 审讯室场景 + 对话 UI 美术化**：像素背景、老人立绘、对话框、真相碎片演出。
4. **D4 · 阶段 0 开场 + 阶段 4 结尾**：幻灯片（街景/旧新闻）+ 结尾字幕。
5. **D5 · 打磨 + 录 demo + 导出网页版（HTML5）**：手感、音效、保底兜底、应对赛事强制要求。

**80% 精力砸在第 1 步**——那是卖点，也是唯一会翻车的地方。

## 技术备注

- **引擎**：Godot 4，导出 HTML5 网页版方便路演/评审点链接即玩。
- **AI**：腾讯云大模型（混元 / 腾讯云上的 DeepSeek）。**密钥不要放进 Godot 客户端**——经一个轻量后端（腾讯云云函数 SCF）代理调用。
- **美术**：itch.io 赛博朋克像素素材包 + 必要处 AI 生成，零原创美术。
- ⚠️ **待确认**：赛事是否**强制**指定某腾讯云产品/模型（详情页动态加载没拉全）。确认后只需替换 AI 接入点，流程不变。
